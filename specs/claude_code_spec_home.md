# Spec: بازطراحی Theme مرکزی + صفحه‌ی Home

این یک دستورالعمل برای Claude Code است. هدف: چیدن یک theme مرکزی و بازنویسی صفحه‌ی Home بر اساس آن.

> قانون کلیدی: هیچ رنگی به‌صورت دستی (hardcoded) داخل صفحه‌ها نوشته نشود. همه‌ی رنگ‌ها از theme مرکزی خوانده شوند، تا آپدیت آینده‌ی UI با تغییر یک فایل ممکن باشد.

---

## بخش ۱ — Theme مرکزی

فایل `lib/shared/theme/app_theme.dart` را بازنویسی کن. این اپ **فقط dark mode** دارد (پس‌زمینه مشکی یکدست). یک کلاس `AppColors` بساز که tokenهای زیر را نگه دارد، و آن‌ها را در `ThemeData` و `ColorScheme` تزریق کن.

### پالت رنگ (dark)

پس‌زمینه و سطوح:
- `background` = `#000000` (پس‌زمینه‌ی اصلی، مشکی یکدست)
- `surface` = `#141414` (کارت‌های خنثی)
- `surfaceVariant` = `#1C1C1C` (آیکون‌بکس‌ها، سطح بالاتر)
- `border` = `#232323` (حاشیه‌ی پیش‌فرض، نازک)

متن:
- `textPrimary` = `#F5F0E8`
- `textSecondary` = `#9A958C`
- `textMuted` = `#6E6E73`

برند:
- `primary` = `#8B7BE8` (بنفش — برای دکمه‌ها، تب فعال، آیتم فعال نوار پایین)

رنگ‌های اولویت تسک (هر اولویت یک رنگ):
- `priorityCritical` (خیلی فوری) = `#E24035` (قرمز)
- `priorityHigh` (فوری) = `#E5A94E` (نارنجی)
- `priorityMedium` (متوسط) = `#D6C535` (زرد)
- `priorityLow` (کم) = `#3F8ADD` (آبی)

عادت‌ها / موفقیت:
- `success` = `#2E8B45` (سبز — تیک عادت‌های انجام‌شده)
- `successAccent` = `#5DCB7A` (سبز روشن — آیکون و متن استریک)

### نکات theme
- فونت: `Vazirmatn` را به‌عنوان فونت اصلی ست کن (برای فارسی). اگر در `pubspec.yaml` نیست، اضافه‌اش کن و راهنمای دانلود فونت را در خروجی بنویس.
- `useMaterial3: true`
- `scaffoldBackgroundColor` = `AppColors.background`
- radius پیش‌فرض کارت‌ها = `16`
- elevation همه‌جا `0` (بدون سایه‌ی Material؛ عمق فقط از طریق border و رنگ سطح)
- یک helper بساز که از روی مقدار اولویت تسک، رنگ متناظر را برگرداند (مثلاً `AppColors.priorityFor(priority)`).

## بخش ۲ — صفحه‌ی Home

فایل `lib/modules/home/screens/home_screen.dart` را بازنویسی کن. ساختار از بالا به پایین:

### ۱) نوار تب (بالا)
- سمت راست: دو تب «تسک‌های امروز» (فعال، با زیرخط بنفش `primary`) و «عادت‌های امروز».
- سمت چپ: یک آیکون فیلتر/تنظیمات (`Icons.tune`).
- تب فعال متن `textPrimary` + زیرخط؛ تب غیرفعال `textMuted`.

> توجه: تقویم بالای صفحه، عکس پروفایل، نام کاربر و جملات انگیزشی **حذف** شوند. صفحه مستقیم با نوار تب شروع شود.

### ۲) لیست تسک‌های امروز
هر تسک یک کارت با این مشخصات:
- رنگ پس‌زمینه و حاشیه‌ی کارت از روی **اولویت** تسک می‌آید (نسخه‌ی خیلی تیره‌ی رنگ اولویت برای پس‌زمینه، نسخه‌ی تیره برای border).
- یک هاله‌ی ملایم گوشه‌ی بالا-چپ کارت با همان رنگ اولویت (با opacity پایین، حدود ۱۳٪). این را با `RadialGradient` بساز، **نه** با blur/BackdropFilter (برای حفظ سرعت).
- چیدمان داخل کارت (راست‌به‌چپ):
  - دایره‌ی چک‌باکس (خالی، border دور) سمت راست‌ترین.
  - عنوان تسک (`textPrimary`, weight 500) + آیکون دسته کنارش.
  - خط دوم: ساعت + نام پروژه/دسته (`textSecondary`, کوچک‌تر).
  - سمت چپ: یک pill با متن اولویت (مثل «خیلی فوری») با رنگ اولویت، و زیرش نقطه‌ها (تعداد نقطه = شدت اولویت: ۳ نقطه critical، ۲ نقطه high، ۱ نقطه medium/low).
- از `ListView.builder` استفاده کن (فقط آیتم‌های قابل‌دیدن رندر شوند).
- empty state: وقتی تسکی نیست، یک آیکون و متن «تسکی برای امروز نداری».

### ۳) جداکننده
یک خط نازک افقی (`border`) بین تسک‌ها و عادت‌ها.

### ۴) لیست عادت‌های امروز
عنوان بخش «عادت‌های امروز» با آیکون سبز. هر عادت یک کارت:
- پس‌زمینه‌ی سبز خیلی تیره (`#0C140C`)، border سبز تیره (`#1E3A1E`).
- سمت راست: آیکون‌باکس سبز + نام عادت + زیرنویس (مثلاً «۲۰ دقیقه»).
- وسط: ردیف دایره‌های هفته (انجام‌شده = پر سبز `success` با تیک سفید؛ انجام‌نشده = خالی با border).
- سمت چپ: استریک با ایموجی 🔥 و تعداد روز، با رنگ `successAccent`.

### ۵) دکمه‌ی افزودن تسک
یک دکمه‌ی تمام‌عرض با border بنفش کم‌رنگ، متن «افزودن تسک» + آیکون +، رنگ `primary`.

## قوانین کلی پیاده‌سازی
- راست‌به‌چپ (RTL) رعایت شود — اپ از قبل `Directionality(rtl)` دارد.
- empty state و loading state و error state برای لیست‌ها مدیریت شوند (طبق Definition of Done در CLAUDE.md).
- داده‌ها از providerهای موجود خوانده شوند (`todayTasksProvider`, `activeHabitsProvider` و غیره). اگر فیلدی مثل «اولویت» در مدل تسک نیست، اول آن را گزارش بده و پیشنهاد بده چطور اضافه شود — خودسرانه schema را تغییر نده.
- بعد از پیاده‌سازی، یک بار `flutter analyze` بزن و خطاها را برطرف کن.

## ترتیب کار
۱. اول `app_theme.dart` و `AppColors` را بساز.
۲. بعد `home_screen.dart` را روی همان بنویس.
۳. اگر مدل تسک «اولویت» ندارد، قبل از ادامه گزارش بده.
